# R11.2 — Subteam modal redesign + copy rewrite

**Status:** planned
**Files:** `src/components/SubteamModal.tsx` (redesign), `src/components/project/ProjectShell.tsx`
(+ AccentContext provider), `src/components/project/accentContext.ts` (new), the 13 subteam
files (→ structured data + rewritten copy), `src/components/StarStat.tsx` (unchanged — still the
trigger card; confirm no change needed)

Subteam files (13): cubesat `subteam-descriptions/{Structures,Thermal,Power,Communications,
SystemsIntegration,FSW,Payload,ADCS}.tsx`; robotics `{Mechanical,Electrical,Software}.tsx`;
weather `{Software,Structures}.tsx`.

## Spec

**Problem:** Each subteam card opens a Radix `Dialog` whose body is a children blob: an intro
paragraph, a `<ul>` of `<li><span class="font-bold">Label:</span> text</li>`, and an "In
essence, the X Team …" closer. Two issues:
(a) **No design** — it's an unstyled wall of text; nothing themes to the project, no hierarchy.
(b) **Copy reads AI-generated** — templated scaffolding across all 13 ("Their scope generally
covers:", "In essence, the … Team transforms/delivers …", hedge words, list-of-everything).

**Approach:**

### 1. Structured data model (replaces the JSX blobs)
Define `interface Subteam { name: string; lead: string; icon: LucideIcon; summary: string;
responsibilities: { title: string; body: string }[] }`. Convert each of the 13 subteam files
from a JSX-children component into one that supplies this data (keep the file-per-subteam layout;
each file exports its `Subteam` object and renders `<SubteamModal subteam={data} />`, OR export
the data object and let the page map it — see step 4). `summary` = 1–2 punchy sentences;
`responsibilities` = the existing bullet points as `{title, body}` pairs (titles already exist as
the bold labels). Lead + icon come from the current `StarStatProps` (headline + Icon).

### 2. Redesigned modal (`SubteamModal.tsx`)
Render a designed dialog (not a text wall), themed by `--accent`:
- **Header:** the subteam `icon` in an accent ring (`border-[var(--accent)]`), the name as
  `DialogTitle` (`text-2xl`), and the lead as a small mono chip "LEAD · <name>" in
  `text-[var(--accent)]`. An accent hairline (`border-[var(--accent)]/30`) under the header.
  Add a `DialogDescription` = the `summary` (also fixes the existing Radix "missing Description"
  a11y warning).
- **Summary:** one lead-in line, `text-white/80`, slightly larger.
- **Responsibilities:** the `responsibilities[]` rendered as an accent-marked definition list —
  each item: a small `var(--accent)` index/tick, a `font-semibold` white title, then a
  `text-white/70` body; comfortable vertical rhythm with subtle `divide-white/10` rules. (Not a
  cramped `<ul>`.) Keep it single-column for readability; the dialog already scrolls.
- Keep `DialogContent` dark; cap width (`sm:max-w-lg`) and height (`max-h-[85vh] overflow-y-auto`).

### 3. Accent across the portal (gotcha)
Radix `Dialog` portals to `document.body`, so the `--accent` set on the ProjectShell root does
**not** reach `DialogContent`. Fix with React context, not DOM cascade:
- `accentContext.ts` — `export const AccentContext = createContext<string>("#ffffff")`.
- `ProjectShell` wraps its subtree in `<AccentContext value={config.accent}>` (the card
  `.accent-wash` already works since cards are in-DOM; this is for the portaled modal).
- `SubteamModal` reads `useContext(AccentContext)` and sets `style={{ '--accent': accent }}` on
  `DialogContent`, so the modal themes to the right project. (No prop-threading through 13 files.)

### 4. Wiring
Simplest path that keeps R11.2 self-contained: each subteam file keeps rendering
`<SubteamModal subteam={data} />` (SubteamModal now takes a `subteam: Subteam` prop instead of
`starStatProps` + `children`, and internally renders the StarStat trigger from
`subteam.{name,icon,lead}` + the redesigned dialog body). The pages are unchanged (they still
render `<Structures/>`, `<Mechanical/>`, …). `StarStat` stays the trigger card (its `.accent-wash`
theming from set 10 is retained). Note: SubteamModal currently mutates `starStatProps.className`
(`+ " h-60"`) — drop that mutation; pass sizing cleanly.

### 5. Copy rewrite (de-AI, all 13)
Rewrite `summary` + each `responsibilities[].body` per `design/brand.md` voice (confident,
concrete, terse; no hedging). Rules: kill templated scaffolding ("generally covers", "In essence",
"transforms … into …"); active voice; concrete nouns over filler; ≤ ~2 lines per body; vary
sentence shape so the 13 don't read from one mold. **Preserve factual scope** — do not invent
specific tools, names, or this-semester specifics (see _set.md Open Question).
Example (Structures, before → after):
- before summary: "The Structures team is responsible for the spacecraft's physical frame—the
  skeleton that holds everything together and ensures survival in the harsh conditions of launch
  and orbit. Their scope generally covers:"
- after summary: "Structures builds the frame that keeps SPICEsat intact — from the violence of
  launch to years in vacuum."
- before item: "Mechanical Design & CAD Modeling: Developing the CubeSat's chassis, brackets, and
  deployables (like solar panels or antennas) to fit within CubeSat standards and payload
  constraints."
- after item: title "Chassis & CAD" / body "Design the chassis, brackets, and deployables to
  hit CubeSat form-factor limits and payload constraints."
