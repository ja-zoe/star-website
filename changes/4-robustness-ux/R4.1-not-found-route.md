# R4.1 — 404 NotFound route

**Status:** tests passing
**Files:** `src/routes/NotFound.tsx` (new), `src/App.tsx`

## Spec

**Problem:** There is no catch-all route. Unknown paths (typos, stale links, the Apache SPA
fallback rewriting every path to index.html) render an empty `<main>` — a blank page between
the navbar and footer. Measured: `/does-not-exist` → `main` innerText empty, title stays the
home default. A recruiting site should never show a blank page.

**Approach:**
1. `src/routes/NotFound.tsx`: a small on-brand page (uses the dark space theme already provided
   by the global chrome). Contents: `<Seo title="Page not found — STAR" path={location.pathname}>`
   (no indexable description needed), an `<h1>` "404", a short line ("This page drifted off into
   the void."), and a primary link back home (`<a href="/">Return to STAR</a>` styled like the
   existing white pill / HoverBorderGradient idiom — resolve against design/components.md
   buttons). Keep it lightweight; reuse existing classes, no new deps.
2. `src/App.tsx`: add `<Route path="*" element={<NotFound />} />` as the last route inside the
   existing `<Routes>`. Lazy-load it like the other routes (`const NotFound = lazy(() =>
   import("./routes/NotFound"))`) to stay consistent with R2.2.

Decision: 404 copy is space-themed but brief; CTA is a single "Return to STAR" home link (the
nav is still present above, so no need to duplicate the full menu). Recorded.

State: none.
