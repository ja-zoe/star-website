# R4.3 — Functional project-page CTAs + typo

## Spec

**Problem:** On all three project pages the "About the Project" pill is a `<p>` styled as a
button (`bg-white text-black rounded-xl cursor-pointer`) but is **not interactive** — no link,
no handler. It looks clickable and does nothing (misleading affordance + not keyboard-focusable).
Also a content typo on RoboticsPage: "TThe Robotics project…".

**Approach:** Each page has, directly below the `WavyBackground` hero, a
`<div className="flex flex-col gap-10 text-center">` that opens with the project description.
1. Add `id="about"` to that description wrapper `<div>` on each page.
2. Convert each "About the Project" `<p>` into `<a href="#about" …>` (same visual classes +
   keep it a focusable anchor; the global focus-visible ring from R3.3 applies). Clicking/Enter
   scrolls to the description (the page already has `scroll-smooth` on `<html>`; reduced-motion
   users get an instant jump via the existing media override).
3. Fix "TThe" → "The" on RoboticsPage.

Decision: the CTA scrolls to the on-page description (`#about`) rather than linking elsewhere —
the content it promises is already on the page just below. Recorded; no Open Question.

State: none.
