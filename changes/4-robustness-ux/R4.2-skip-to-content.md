# R4.2 — Skip-to-content link

**Status:** tests passing
**Files:** `src/App.tsx`, `src/index.css`

## Spec

**Problem:** No skip link. Keyboard and screen-reader users must tab through the entire navbar
(logo + 5 nav items + Projects submenu) on every page before reaching content (WCAG 2.4.1
Bypass Blocks). The site already has a single `<main>` landmark (set 1) — it just needs a
reachable skip target.

**Approach:**
1. `src/App.tsx`: add `id="main-content"` to the existing `<main>`. As the very first focusable
   element in the tree (before `<Navbar/>`), render
   `<a href="#main-content" className="skip-link">Skip to main content</a>`.
2. `src/index.css`: style `.skip-link` to be visually hidden until focused, then appear pinned
   top-left over the dark theme (WCAG-compliant — present in DOM/tab order, not display:none):
   ```css
   .skip-link {
     position: fixed; top: 0; left: 0; z-index: 100;
     transform: translateY(-150%);
     background: #fff; color: #000; padding: 8px 16px; border-radius: 0 0 6px 0;
     transition: transform 150ms ease;
   }
   .skip-link:focus { transform: translateY(0); outline: 2px solid #fff; }
   ```
   (Respect reduced-motion: the existing global `@media (prefers-reduced-motion: reduce)` in
   App.css already zeroes transition-duration, so the link snaps in instead of sliding — fine.)

Decision: skip link targets `#main-content`; visually hidden via transform (not display:none) so
it stays in the tab order. Recorded.

State: none.
