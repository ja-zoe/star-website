## Spec

**Problem:** There is no single source of truth for the visual system. Colors are ad-hoc
(`red-400`, `#9D2626`, `#0091ff`, `#03d011`, shadcn oklch neutrals), and CONTEXT.md promised a
`design/` directory that doesn't exist yet. Future visual work (and this set's R3.2/R3.3) has
nothing to resolve against.

**Approach:** Run the design-brand framework (`.claude/skills/design-brand/`) in **auto** mode
(resolution chain: config → auto-detect → default; no questionnaire — this is an autonomous
session). Auto-detect the established identity from the codebase and write it verbatim as the
baseline (tier: system). Detected facts to encode:
- **Color:** surface black `#000`; text white `#fff`; brand accents — STAR red `--color-starred
  #9D2626` (+ the brighter UI red `red-400 #f87171` used on eboard ring), weather-balloon blue
  `--color-wb #0091ff`, robotics green `#03d011`; shadcn neutral ramp (oklch) for component
  chrome. TextHoverEffect rainbow stops (yellow/red/blue/cyan/violet 500s) are a one-off accent.
- **Type:** display+body+mono all = **Space Mono** (loaded in index.html, `.space-mono` on the
  app root). Inter is imported in App.css but effectively unused — note as legacy.
- **Icons:** lucide-react, line style (CONTEXT invariant: lucide only).
- **Components:** shadcn (new-york) over Radix primitives + Aceternity/magicui effects;
  `--radius 0.625rem`. `components-library` = shadcn-new-york → subsumes `components`,
  radius/shadow; Aceternity effects are bespoke on top.
- **Motion:** framer-motion; custom `shimmer`/`moveJawn`; all gated by prefers-reduced-motion
  (set 1, R1.4) — record that as the standing motion rule.
- **Modes:** the app is dark-only (forces the dark space theme); shadcn light tokens exist but
  are unused. Record dark as the single active mode.

Write the six files per `.claude/skills/design-brand/references/output-format.md`. Record
provenance in `_index.md` (mostly `autodetect`, `default` for unspecified slots). This is the
durable record; R3.2/R3.3 and all later visual work resolve against `tokens.md` /
`accessibility.md` / `components.md`.

Decision: encode the existing system as-is (refine-not-reinvent). Where two reds coexist
(#9D2626 brand vs red-400 ring) document both in the accent ramp rather than forcing one, to
avoid an unrequested visual change. Recorded; no Open Question.
