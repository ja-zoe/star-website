# R3.2 — Fix TextHoverEffect radialGradient console errors

**Status:** tests passing
**Files:** `src/components/ui/text-hover-effect.tsx`

## Spec

**Problem:** Every home-page load logs 10 console errors (measured baseline; confirmed still
present after sets 1–2): `<radialGradient> attribute cx: Expected length, "undefined"` and the
matching `cy`. `TextHoverEffect` is used for the big section headings (WELCOME, OUR PROJECTS,
FAQ, JOIN STAR, MEET E-BOARD), so the errors fire ~5× each per load. Cause: the
`<motion.radialGradient id={revealMaskId} animate={maskPosition} …>` has **no initial `cx`/`cy`
attribute** — on first paint, before framer-motion applies `animate`, the element renders with
`cx`/`cy` = undefined, which is an invalid SVG length.

**Approach:** Give the motion.radialGradient a valid initial value for the animated attributes
so the first paint is well-formed. Add `initial={{ cx: "50%", cy: "50%" }}` (matching the
`maskPosition` initial state) to the `<motion.radialGradient>`. framer then animates from that
valid 50%/50% to the cursor-driven `maskPosition`. No visual change (it already starts centered).

Decision: use `initial` (not static `cx`/`cy` attributes) to avoid a static-attr vs
animated-value conflict in framer-motion. Recorded.
