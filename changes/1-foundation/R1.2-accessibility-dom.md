## Spec

**Problem:** Multiple a11y + DOM-validity defects measured in the audit:
- Logo `alt=""` (Navbar:28); footer social icons `alt=""`; JoinUs discord img no alt → 8 imgs
  without alt on home (measured).
- Footer mail link `href=""` + `target="_blank"` → empty/broken link (measured emptyLinks).
- Mobile menu trigger is a bare lucide `<Menu>` SVG, not a focusable labelled control.
- EboardSection `.map()` has no `key` → React console error.
- Invalid DOM nesting → 2 console errors: `<div>` (FlipWords) inside `<p>` (HeroSection);
  `<a>` (inner link) inside `<a>` (HoverBorderGradient `as="a"`) in Hero ("Learn More") and
  AboutStar ("Become a Member"/learn-more uses).

**Approach:**
1. **Navbar.tsx:** logo `alt="STAR — Space Technology Association of Rutgers logo"`. Wrap the
   mobile `<Menu>` in a real `<button type="button" aria-label="Open menu"
   className="text-white">` as the `SheetTrigger asChild` child (keep size 40). Change the
   outer `<div>` root to `<header>` and the nav region semantics: desktop `NavigationMenu`
   already renders a `<nav>`; ensure mobile `<nav>` (already present). Logo `href="/#"` →
   `href="/"` (cleaner home link; section anchors still work).
2. **Footer.tsx:** root `<div>` → `<footer>`. Add alt text: discord "STAR Discord",
   instagram "STAR Instagram", linkedin "STAR LinkedIn". Replace the empty mail anchor with
   `<a href="mailto:rutgersstar@gmail.com" aria-label="Email STAR">` (decision: use a mailto;
   see Notes for address rationale) and drop `target="_blank"` on the mailto. Give the `Mail`
   icon `aria-hidden` (decorative; the link is labelled). Add `aria-label` to each social
   anchor as well since the imgs are inside links.
3. **JoinUsSection.tsx:** discord img `alt="STAR Discord logo"`; wrap link add
   `aria-label="Join the STAR Discord"`.
4. **EboardSection.tsx:** add `key={member.name}` to the mapped `<div>`.
5. **HeroSection.tsx (div-in-p fix):** the paragraph wraps `FlipWords` which renders a
   `<div>`. Change the outer `<p>` to a `<div>` styled identically (keep the same text
   classes) OR render FlipWords inline. Decision: convert the wrapper to a `<div>` with the
   same typography classes and `inline`/flex layout so the sentence still reads on one line;
   keep an inner `<span>` for the static text. Verify the FlipWords visual is unchanged.
6. **HoverBorderGradient (a-in-a fix):** component accepts `as` (defaults to `button`) and is
   used `as="a"` then a child `<a href=...>` is nested inside → nested anchors. Fix at the
   call sites: pass `as="a"` AND the `href` directly to HoverBorderGradient (remove the inner
   `<a>`), so only one anchor renders. Confirm HoverBorderGradient forwards arbitrary props
   (href) to the rendered tag; if it doesn't, extend it to spread `...rest` onto the element.
   Apply to HeroSection "Learn More" (`href="#AboutStarSection"`) and any AboutStar usage.

State: none. Pure markup/attribute changes.
