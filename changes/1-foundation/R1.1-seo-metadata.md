## Spec

**Problem:** Every route serves `<title>Star</title>` (measured identical on all 4 routes),
no meta description, no Open Graph / Twitter card, favicon declared `type="image/svg+xml"`
but the file is a PNG (wrong MIME), and there are no semantic landmarks. For a public club
recruiting site, discoverability and link-preview cards matter and are currently absent.

**Approach:**
1. **`index.html` static defaults** (the head a crawler/social bot sees before JS runs, and
   the static-host baseline):
   - `<title>STAR — Space Technology Association of Rutgers</title>`
   - `<meta name="description" content="...">` (~150 chars: Rutgers' premier space & technology
     club — CubeSat, Robotics (NASA Lunabotics), and high-altitude weather balloon projects.
     Open to all majors, no experience required.)
   - Open Graph: `og:type=website`, `og:site_name=STAR`, `og:title`, `og:description`,
     `og:image=/star-image.png`, `og:url`. Twitter: `twitter:card=summary_large_image`,
     `twitter:title`, `twitter:description`, `twitter:image`.
   - `<meta name="theme-color" content="#000000">`.
   - Fix favicon: `<link rel="icon" type="image/png" href="/star-logo-transparent.png" />`
     (correct MIME). Add `<link rel="apple-touch-icon" href="/star-logo-transparent.png" />`.
2. **`src/components/Seo.tsx`** — dependency-free per-route head manager relying on React 19's
   native hoisting of `<title>`/`<meta>` rendered in the component tree. Props:
   `{ title: string; description?: string; path?: string }`. Renders:
   `<title>{title}</title>`, `<meta name="description">`, `<meta property="og:title">`,
   `<meta property="og:description">`, `<meta property="og:url">`, `<meta name="twitter:title">`,
   `<meta name="twitter:description">`. Title format: page-specific string already includes
   the brand (e.g. "Robotics — STAR"). No external libs (CONTEXT: React 19 hoisting).
3. **Per-route `<Seo>`** at the top of each page component:
   - Home: `title="STAR — Space Technology Association of Rutgers"`,
     desc = the index.html description, `path="/"`.
   - Cubesat: `title="CubeSat — STAR"`, desc about the CubeSat satellite team, `path="/cubesat"`.
   - Robotics: `title="Robotics — STAR"`, desc about NASA Lunabotics rover team, `path="/robotics"`.
   - Weather Balloon: `title="Weather Balloon — STAR"`, desc about high-altitude balloon team,
     `path="/weather-balloon"`.
   Use `window.location.origin` at render for `og:url` base (client-only; fine for a SPA — the
   static index.html already carries an absolute-ish default).
4. **Semantic landmarks** (also serves a11y, but the markup lives here): in `App.tsx` wrap
   `<Routes>` output in `<main>`; `Navbar` root becomes `<header>`/`<nav>`, `Footer` root
   becomes `<footer>` (done in R1.2 — coordinate: R1.2 owns Navbar/Footer tags, R1.1 owns the
   `<main>` wrapper in App.tsx). To avoid collision, **R1.1 only adds `<main>` in App.tsx**;
   nav/footer tag changes belong to R1.2.

State: none persisted. Everything is render-time DOM.
