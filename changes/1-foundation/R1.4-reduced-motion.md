## Spec

**Problem:** The site is animation-heavy (twinkling stars canvas, two shooting-star canvases,
tsparticles sparkles, cobe globe auto-rotation, framer flip-words, CSS shimmer/moveJawn/pulse)
with zero `prefers-reduced-motion` handling. Users with vestibular sensitivity / the OS
"reduce motion" setting get the full motion load — an a11y gap (WCAG 2.3.3) and a perf/jank
liability on low-end devices.

**Approach:** Two layers.

1. **CSS layer — `src/App.css`** add at end:
   ```css
   @media (prefers-reduced-motion: reduce) {
     *, *::before, *::after {
       animation-duration: 0.001ms !important;
       animation-iteration-count: 1 !important;
       transition-duration: 0.001ms !important;
       scroll-behavior: auto !important;
     }
     .moveJawn, .animate-shimmer, .animate-pulse { animation: none !important; }
   }
   ```
   Also remove `scroll-smooth` jump by the scroll-behavior override (index.html has
   `scroll-smooth` on `<html>`). Keep accordion (functional, 0.2s) — the global rule reduces
   it harmlessly.

2. **JS/canvas layer — `src/hooks/usePrefersReducedMotion.ts`** (new): subscribe to
   `window.matchMedia('(prefers-reduced-motion: reduce)')`, return boolean, update on change.
   SSR-safe guard not needed (client-only SPA) but guard `typeof window`.
   Use it to short-circuit the expensive loops:
   - **stars-background.tsx:** when reduced, render the stars once (static field) and skip the
     twinkle `requestAnimationFrame` loop (keep the painted stars so the bg still looks right).
   - **shooting-stars.tsx:** when reduced, render nothing / never schedule a new star
     (return early before the spawn timer).
   - **sparkles.tsx:** when reduced, lower particle count drastically or disable the move
     (`particles.move.enable=false`) so it's a static starfield rather than drifting.
   - **globe.tsx (cobe):** when reduced, stop the auto-rotation (`phi` stays constant; keep
     pointer-drag if present) so the globe is still shown but not spinning.
   - **flip-words.tsx:** when reduced, render only the first word statically (no cycling
     animation), so the sentence still completes ("...space and technology enthusiasts").

Decision: reduced-motion keeps the *visual identity* (stars still visible, globe still shown)
but removes continuous motion — refine, don't strip. Recorded here; no Open Question.

State: none (matchMedia is the source of truth).
