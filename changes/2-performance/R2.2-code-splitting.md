# R2.2 — Code splitting (lazy routes + defer three.js)

**Status:** tests passing
**Files:** `src/App.tsx`, `src/components/ProjectCard.tsx`, possibly
`vite.config.ts` (manualChunks, optional)

## Spec

**Problem:** The app ships a single 1,448 kB JS chunk (gzip 423 kB) — Vite warns it exceeds
500 kB. Everything loads up front, including `three` + `@react-three/fiber` (~600 kB) pulled
in by `CanvasRevealEffect`, which only renders on project-card hover, and the three project
route pages, which a home-page visitor may never open.

**Approach:**
1. **Route-level code splitting (App.tsx):** wrap the four page components in `React.lazy()`
   (`const HomePage = lazy(() => import("./routes/home/HomePage"))`, etc.) and wrap `<Routes>`
   in `<Suspense fallback={...}>`. Fallback: a minimal centered spinner or just `null` over the
   existing black/stars background (the global StarsBackground/Footer stay mounted outside
   Suspense, so the page never flashes white). Each route becomes its own chunk; project pages
   (WavyBackground/simplex-noise) split out of the home/initial chunk.
2. **Defer three.js (ProjectCard.tsx):** `CanvasRevealEffect` is statically imported, forcing
   three into the initial chunk. Convert it to `const CanvasRevealEffect = lazy(() =>
   import("./ui/canvas-reveal-effect").then(m => ({ default: m.CanvasRevealEffect })))` and
   render it inside `<Suspense fallback={null}>` only within the existing `{hovered && ...}`
   branch. Result: three.js loads as a separate chunk on first card hover, off the critical
   path. Keep the hover UX identical (the reveal already animates in on hover).
3. **(Optional) vite manualChunks:** only if a quick win — split `react`/`react-dom`/
   `react-router` into a `vendor` chunk for cacheability. Skip if it complicates the build;
   the lazy boundaries above are the substantive fix.

Decision: fallback is `null` (not a spinner) because the stars background + footer remain
visible during the brief chunk fetch, so a spinner would add visual noise. Recorded.

State: none.
