# R8.1 — Security/hardening response headers

**Status:** tests passing
**Files:** `public/.htaccess`

## Spec

**Problem:** The static host sends no defensive response headers. Standard low-risk hardening
headers (MIME-sniffing protection, clickjacking protection, referrer privacy, feature lockdown)
are missing on a public recruiting site.

**Approach:** Add one `<IfModule mod_headers.c>` block to `public/.htaccess` (a second
mod_headers block is fine; or extend the existing one). Set, for all responses:
- `X-Content-Type-Options: nosniff` — stop MIME-type sniffing.
- `X-Frame-Options: SAMEORIGIN` — block framing/clickjacking (site isn't embedded anywhere).
- `Referrer-Policy: strict-origin-when-cross-origin` — sensible referrer privacy default.
- `Permissions-Policy: geolocation=(), camera=(), microphone=(), interest-cohort=()` — disable
  unused powerful features + opt out of FLoC.
All inside `<IfModule mod_headers.c>` so a host without mod_headers skips it (no 500).
Add a commented CSP scaffold + note explaining it's deferred (inline styles/fonts need tuning).
Do NOT touch the existing caching/compression/SPA/ErrorDocument rules.

Decision: ship only universally-safe headers; CSP deferred with a commented starting point.
Recorded; no Open Question.

State: none (Apache directives; still fully static).
