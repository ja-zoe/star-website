# R7.1 — Optimized favicons

**Status:** planned
**Files:** `scripts/optimize-images.mjs` (extend), `public/icons/*.png` (new, generated),
`index.html` (icon links), `public/site.webmanifest` (icon paths)

## Spec

**Problem:** The favicon is `star-logo-transparent.png` (747×737, **192 KB**), referenced for
`rel=icon`, `apple-touch-icon`, and both manifest icons. Every visit downloads ~192 KB for a
≤32px tab icon — pure waste.

**Approach:** Extend `scripts/optimize-images.mjs` to also emit, from
`public/star-logo-transparent.png`, optimized PNGs into `public/icons/`:
`favicon-32.png` (32×32), `icon-192.png` (192×192), `icon-512.png` (512×512), all `fit:contain`
on transparent background, compressed (sharp png `compressionLevel:9`). Then:
- `index.html`: `rel="icon"` → `/icons/favicon-32.png` (32×32, type image/png) plus an
  `apple-touch-icon` → `/icons/icon-192.png`.
- `public/site.webmanifest`: icons → `/icons/icon-192.png` (192) and `/icons/icon-512.png` (512).
Keep `public/star-logo-transparent.png` (still the manifest/source logo + used elsewhere).

Decision: PNG (not .ico) — modern browsers accept PNG favicons; keeps it in the sharp pipeline.
Sizes 32/192/512 cover tab + PWA. Recorded.

State: none.
