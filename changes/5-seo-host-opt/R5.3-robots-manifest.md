## Spec

**Problem:** No `robots.txt` (no explicit crawl policy) and no web app manifest (no installable
metadata, no standardized icon/name/theme for add-to-homescreen or richer browser UI).

**Approach:**
1. **`public/robots.txt`** — allow all crawlers:
   ```
   User-agent: *
   Allow: /
   ```
   No `Sitemap:` line (deploy domain unknown; would require an absolute URL). A short comment
   explains the omission so a maintainer can add it once the domain is fixed.
2. **`public/site.webmanifest`** — minimal valid manifest:
   `name` "STAR — Space Technology Association of Rutgers", `short_name` "STAR",
   `description`, `start_url` "/", `display` "standalone", `background_color` "#000000",
   `theme_color` "#000000", `icons` → `/star-logo-transparent.png` (192 & 512 `purpose:any`,
   type image/png — the existing logo; acceptable as a single source, browsers scale).
3. **`index.html`** — add `<link rel="manifest" href="/site.webmanifest" />` (theme-color meta
   already added in set 1).

Decision: robots allow-all, no sitemap line (domain unknown); manifest reuses the existing PNG
logo rather than generating new icon sizes (keeps scope tight, no new binary assets). Recorded.
