# R5.2 — JSON-LD Organization structured data

**Status:** tests passing
**Files:** `index.html`

## Spec

**Problem:** No structured data. Search engines have no machine-readable description of STAR as
an organization (name, parent org, social profiles, logo), which limits rich-result eligibility
and entity understanding for a recruiting club.

**Approach:** Add a single `<script type="application/ld+json">` block in `index.html` `<head>`
describing STAR as a `CollegeOrUniversity`-affiliated organization. Use schema.org
`Organization` (with `parentOrganization` Rutgers / SEDS, where factual) — domain-independent:
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "STAR — Space Technology Association of Rutgers",
  "alternateName": "STAR",
  "description": "Rutgers' premier space & technology club: CubeSats, NASA Lunabotics robotics, and high-altitude weather balloons.",
  "logo": "/star-logo-transparent.png",
  "email": "rutgersstar@gmail.com",
  "parentOrganization": [
    { "@type": "CollegeOrUniversity", "name": "Rutgers University" },
    { "@type": "Organization", "name": "Students for the Exploration and Development of Space (SEDS)" }
  ],
  "sameAs": [
    "https://discord.gg/vHa52wx9VK",
    "https://www.instagram.com/ruspacetech/",
    "https://www.linkedin.com/company/spacetechru/"
  ]
}
```
No absolute site `url` (deploy domain unknown — omitting is valid; logo is a root-relative path
which scrapers resolve against the page origin). Values are all facts already in the codebase
(email from git history, the three social links from Footer).

Decision: `@type: Organization` with `parentOrganization` Rutgers + SEDS; omit `url`; reuse the
verified email + social links. Recorded; no Open Question.
