# R13.2 — Scope the scroll underlay to the nav controls only

**Status:** tests passing
**Files:** `src/components/Navbar.tsx`, `design/components.md`

## Spec

**Problem (user follow-up to R13.1):** R13.1 applied the scrolled underlay to the whole
`<header>` (full-width `bg-black/80` bar + bottom border). The user wants only the **navbar
portion with the links** to get the underlay — not the whole header — so the logo and the empty
space stay transparent.

**Approach:** Move the underlay off the `<header>` onto the nav controls.
- Header reverts to a plain transparent full-width container (no scrolled bg/border).
- Extract the scroll styling into a shared `underlay` class string:
  `cn("transition-colors duration-300 border border-transparent", scrolled && "bg-black/80
  backdrop-blur-md border-white/10")`. Border is transparent at the top → coloured on scroll, so
  toggling causes **no layout shift**.
- Apply `underlay` to:
  - the desktop link cluster `NavigationMenuList` (+ `rounded-full px-3 py-1.5`) → a pill behind
    just the links, and
  - the mobile menu button (+ `rounded-lg p-1`) → a contained chip behind the hamburger.
- Logo `<a>` untouched (always transparent). `scrolled` state + listener unchanged from R13.1.
