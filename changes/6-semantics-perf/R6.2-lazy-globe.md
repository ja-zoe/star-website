# R6.2 — Lazy-mount the globe (IntersectionObserver)

**Status:** planned
**Files:** `src/components/magicui/globe.tsx`

## Spec

**Problem:** The home page globe (cobe WebGL) initializes and runs its `requestAnimationFrame`
render loop on page load, even though it lives far below the fold (after the About text +
`mb-[700px]` spacer). Visitors who never scroll there still pay the WebGL init + continuous
render cost from first paint.

**Approach:** Defer `createGlobe` until the canvas container is near the viewport, via
`IntersectionObserver`.
- Add an `inView` state (default false) and observe the existing wrapper `<div ref>` (add a
  ref to the outer container). Observer with `rootMargin: "200px"` (start loading just before
  it scrolls in) sets `inView=true` once and disconnects.
- The `useEffect` that calls `createGlobe` gains `inView` as a guard/dep: it only initializes
  once `inView` is true. Keep the existing reduced-motion behavior (R1.4): when reduced, the
  globe still mounts (when in view) but doesn't auto-rotate.
- Keep the fade-in (`opacity` 0→1) on init so it appears smoothly when it enters.
- SSR/guard: `typeof IntersectionObserver !== "undefined"` (client-only SPA, but guard
  defensively); if unavailable, fall back to `inView=true` immediately (graceful).

Decision: rootMargin 200px so the globe is ready by the time it's on-screen (no pop-in delay);
observe-once then disconnect. Reduced-motion handling unchanged. Recorded; no Open Question.

State: `inView` local state in Globe.
